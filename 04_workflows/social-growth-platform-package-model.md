# Social Growth Platform Package Model
Workflow · Public-safe packaging model for all-social and platform-specific systems

## Why this exists
Use this model to keep `prompt-library` broadly useful and free while creating a separate social-playbook product with premium depth.

## Core Structure

## 1) Main Package: "All-Social Growth OS"
A unified package that answers:
- What each platform is for
- How resources are split
- How strategy, copy, retention, engagement, and analytics connect

Recommended persona stack:
- Social Growth Lead
- Content Strategist
- Post Writer
- Engagement Engine
- Signal Reader

## 2) Platform Packages: "Instagram", "Threads", "TikTok" (and future)
Each platform package adds specialist layers:
- Instagram: growth strategist, viral copywriter, faceless architect, retention optimizer, brand positioning editor
- Threads: Threads Growth Strategist + copy and engagement routing
- TikTok: TikTok Growth Strategist + retention and test operator routing

## Packaging Rule
- Keep framework-level assets in `prompt-library`.
- Keep advanced scoring, benchmark logic, and commercialization playbooks in the private project.

## Traffic Strategy (public-safe)
- Publish free, high-utility persona interfaces and workflow skeletons in `prompt-library`.
- Use examples and quick-start workflows to demonstrate outcomes.
- Point users toward deeper implementation layers managed outside the public repo.

## 90-Day Build Sequence
1. Build the all-social base package and routing standards.
2. Build Instagram package to mature depth and QA.
3. Add Threads package.
4. Add TikTok package.
5. Add quarterly review rubric for cross-platform reallocation.

## Success Criteria
- Users can run cross-platform strategy without confusion.
- Platform-specific workflows are modular but interoperable.
- Public docs remain useful without exposing proprietary methods.
