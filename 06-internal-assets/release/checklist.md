# Release Checklist

## Pre-Release

- [ ] README links and counts are accurate
- [ ] New files follow naming and format conventions
- [ ] No unresolved placeholders remain
- [ ] Structural changes are reflected in docs

## Quality Gate

- [ ] Run repository checks (link/reference/manual sanity)
- [ ] Spot-check representative personas/tasks/workflows
- [ ] Validate any renamed paths are updated everywhere

## Release Notes

- [ ] Summarize what changed
- [ ] Note any migration steps
- [ ] Call out future-facing TODOs separately
