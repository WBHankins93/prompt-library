# Contribution Policy

## Scope

This repository is open to contributions, but all additions must preserve the architecture and quality bar.

## Required for New Content

1. Match directory purpose (`00`-`05` runtime layers, `06` operational assets).
2. Follow naming and frontmatter conventions for the target content type.
3. Include clear intent, expected outputs, and practical examples where applicable.
4. Avoid duplicating standards that are already defined in foundation or response standards.
5. Keep changes narrowly scoped and easy to review.

## Review Criteria

Contributions are evaluated on:
- structural alignment with load order
- clarity and actionability
- non-redundancy
- consistency with existing style
- long-term maintainability

## Change Safety

For substantial edits (renames, re-organization, taxonomy changes), include a migration note in the PR description.
