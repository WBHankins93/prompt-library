# Naming Policy (v1)

## Directory Numbering

- `00`-`05`: runtime prompt assets
- `06`: operational/internal assets

## File Naming

- Use lowercase kebab-case for file names.
- Keep names descriptive and stable.
- Prefer one concept per file.

## Persona Naming

- Persona display names should be concise and unambiguous.
- Canonical persona title appears once in frontmatter and H1.
- Alternative labels belong in descriptive text, not as canonical title variants.

## Workflow and Task Naming

- Name by outcome, not by internal implementation.
- Keep titles understandable to non-technical users.

## Rename Rule

If a rename is needed, update references in READMEs and roadmap docs in the same change set.
